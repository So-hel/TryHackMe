## Task 1 — How Websites Work

When you visit a website, your browser (like Safari or Google Chrome) makes a **request** to a web server asking for information. The server responds with data that your browser uses to render the page.

There are two major components that make up a website:

| Component | Description |
|-----------|-------------|
| **Front End (Client-Side)** | The way your browser renders a website |
| **Back End (Server-Side)** | A server that processes your request and returns a response |


### ✅ Q1: What term best describes the component of a web application rendered by your browser?

**Answer:** `Front end`

## Task 2 — HTML

Websites are primarily created using:

- **HTML** — builds websites and defines their structure
- **CSS** — makes websites look pretty by adding styling options
- **JavaScript** — implements complex, interactive features on pages

**HyperText Markup Language (HTML)** is the language websites are written in. Elements (also known as tags) are the building blocks of HTML pages.

### 🔹 HTML Document Structure

```html
<!DOCTYPE html>
<html>
  <head>
    <title>Page Title</title>
  </head>
  <body>
    <h1>My Heading</h1>
    <p>My paragraph.</p>
  </body>
</html>
```

### 🔹 Key HTML Elements

| Tag | Purpose |
|-----|---------|
| `<!DOCTYPE html>` | Defines the page as HTML5 |
| `<html>` | Root element of the HTML page |
| `<head>` | Contains metadata about the page (e.g. title) |
| `<body>` | Defines the document body — only this is shown in the browser |
| `<h1>` | Defines a large heading |
| `<p>` | Defines a paragraph |
| `<button>` | Defines a clickable button |
| `<img>` | Embeds an image |

### 🔹 HTML Attributes

Tags can contain **attributes** to add extra information or styling:

```html
<!-- class attribute for styling -->
<p class="bold-text">Hello</p>

<!-- src attribute for images -->
<img src="img/cat.jpg">

<!-- id attribute — must be unique per element -->
<p id="example">Unique paragraph</p>

<!-- multiple attributes -->
<p attribute1="value1" attribute2="value2">Content</p>
```

> 💡 **Tip:** You can view the HTML of any website by right-clicking and selecting **"View Page Source"** (Chrome) / **"Show Page Source"** (Safari).

## Task 3 — JavaScript

**JavaScript (JS)** is one of the most popular coding languages in the world. It allows pages to become **interactive** and dynamic. Without JavaScript, a page would always be static.

JS can:
- Dynamically update the page in real-time
- Change the style of elements on events (e.g. button click)
- Display moving animations

### 🔹 Adding JavaScript

```html
<!-- Inline within script tags -->
<script>
  document.getElementById("demo").innerHTML = "Hack the Planet";
</script>

<!-- Loaded from an external file -->
<script src="/location/of/javascript_file.js"></script>
```

### 🔹 JavaScript Events

HTML elements can have events like `onclick` or `onhover` that execute JavaScript:

```html
<!-- onclick event on a button -->
<button onclick='document.getElementById("demo").innerHTML = "Button Clicked";'>
  Click Me!
</button>
```

## Task 4 — Sensitive Data Exposure

**Sensitive Data Exposure** occurs when a website doesn't properly protect (or remove) sensitive clear-text information from the end-user — usually found in a site's **frontend source code**.

A developer may have forgotten to remove:
- 🔑 Login credentials
- 🔗 Hidden links to private parts of the website
- 📄 Other sensitive data in HTML or JavaScript comments

### 🔹 Example

```html
<!-- TODO: remove before going live -->
<!-- username: admin | password: testpasswd -->
```

> ⚠️ **Security Note:** Whenever assessing a web application, one of the **first things** you should do is review the page source code for exposed credentials or hidden links.

## Task 5 — HTML Injection

**HTML Injection** is a vulnerability that occurs when **unfiltered user input** is displayed on the page.

If a website fails to **sanitise** (filter) user input, an attacker can inject HTML or JavaScript code into the page — allowing them to control the page's appearance and functionality.

### 🔹 How it Works

```
User Input → Not Sanitised → Displayed on Page → Attacker Controls Page
```

> ⚠️ **Key Takeaway:** Always sanitise and validate user input on both the client-side and server-side to prevent HTML/JS injection attacks.
